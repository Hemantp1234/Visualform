🔐 SECURITY ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════

Master Key System:
  User Password + Username
         ↓
  PBKDF2 (100,000 iterations)
         ↓
  32-byte Master Key (unique per user)
         ↓
  Fernet Encryption
         ↓
  AWS credentials encrypted at rest

Even if database is stolen:
  ✅ AWS keys remain encrypted
  ✅ Attacker needs user password
  ✅ PBKDF2 prevents brute force
  ✅ Defense in depth approach

✨ HOW IT REPLACES TERRAFORM
═══════════════════════════════════════════════════════════════════════════

Terraform Way:
  terraform plan
  terraform apply
  Wait & review
  Check .tfstate
  terraform destroy

VisualForm Way:
  Fill web form
  Click "Launch Instance"
  Instance appears in dashboard
  Click "Terminate"
  Done!

Benefits:
  ✅ No code to learn (GUI instead of HCL)
  ✅ Instant feedback (no plan step)
  ✅ Live state (always accurate)
  ✅ Perfect for testing/demos
  ✅ Built-in multi-user support
